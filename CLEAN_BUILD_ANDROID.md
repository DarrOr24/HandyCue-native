# Clean Android Build Guide

## Prerequisite: Java (JDK 17)

Android builds require Java. If you see **"Unable to locate a Java Runtime"**, install JDK 17 first.

### Install Java on macOS (Homebrew)

```bash
brew install --cask zulu@17
```

Then add to your `~/.zshrc` (or `~/.bash_profile`):

```bash
export JAVA_HOME=/Library/Java/JavaVirtualMachines/zulu-17.jdk/Contents/Home
export PATH="$JAVA_HOME/bin:$PATH"
```

If the path differs (e.g. on Apple Silicon), run `brew info zulu@17` to find the install location.

Reload your shell: `source ~/.zshrc`

Verify: `java -version` should show Java 17.

---

## Prerequisite: Android SDK

The build needs the Android SDK. If you see **"SDK location not found"** or **"ANDROID_HOME"**, install Android Studio (which includes the SDK):

1. Download: https://developer.android.com/studio
2. Install and open Android Studio
3. Go to **Settings → Appearance & Behavior → System Settings → Android SDK** and ensure the SDK is installed (default: `~/Library/Android/sdk`)
4. Add to `~/.zshrc`:
   ```bash
   export ANDROID_HOME=$HOME/Library/Android/sdk
   export PATH=$PATH:$ANDROID_HOME/platform-tools:$ANDROID_HOME/emulator
   ```
5. Reload: `source ~/.zshrc`

## Steps for a Clean Android Build

The `RNGestureHandlerModule could not be found` error usually means the native Android binary is out of sync with your JavaScript dependencies. A clean prebuild fixes this.

### 1. Stop any running Metro bundler
Press `Ctrl+C` in the terminal where Expo is running.

### 2. Remove generated native folders and caches
```bash
cd /Users/darror/Projects/Handycue/HandyCue-native

# Remove Android native project (it will be regenerated)
rm -rf android

# Clear Expo cache
rm -rf .expo

# Optional: clear Metro cache
rm -rf /tmp/metro-* 2>/dev/null || true
```

### 3. Regenerate native project
```bash
npx expo prebuild --clean --platform android
```
This recreates the `android` folder with all native modules (including gesture handler) properly linked.

### 4. Build and run
```bash
npx expo run:android
```

## If it still fails

### Full nuclear option
```bash
# 1. Remove everything
rm -rf android node_modules .expo

# 2. Reinstall dependencies
npm install

# 3. Regenerate native project
npx expo prebuild --clean --platform android

# 4. Run
npx expo run:android
```

### Why this happens
- `react-native-gesture-handler` (used by `react-native-draggable-flatlist` in CueCraft) requires native code
- The `android` folder is generated by `expo prebuild` and is in `.gitignore`
- If the native project was built before a dependency change, or caches are stale, the native binary can miss modules
- `expo prebuild --clean` regenerates the project from scratch with the current dependencies
